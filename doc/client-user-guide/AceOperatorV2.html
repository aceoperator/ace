Your Operator Login Access

To access Ace Operator and login as an operator, use your web browser to navigate to your operator login page as previously sent to you by QUIK Computing or to a page your webmaster has created with a similar button pointing to the V2 URL for your operator group.

Here is an example of the operator login access page:


Click on the Login button/icon to log into the system.

Logging In as an Operator

When you click on the login icon, you should get a new window with a login Username/Password prompt. This new window will serve your entire login session including all active chats and popups - so it is the only window that you need to keep open while using Ace Operator. You don't have to keep the login access page with the button (above) open.

Enter the login user name and password for one of your operators and click “Login”. When successfully logged in, you'll come to your operator's information screen:



This screen contains the user (operator's) information and settings and is explained in more detail later.

The important point is that everything happens within this browser window, on the various tabs: My Info, Conversations, Contacts, Chat (not shown yet - you won't see the Chat tab until you initiate or receive a call) and the Logout tab. You can get to the different areas of information by clicking on the tabs. The tabs of most interest are the Conversations tab which shows the list of chats, the Chat tab which shows a selected chat so you can read/write to it, and the Contacts tab which shows other members of your group who are logged in. These are all discussed in detail below.

Notifications

The new version of Ace Operator has audio notifications as before. In addition, for visual notification, if something happens as a result of somebody else's action and you are not on the tab corresponding to that event, you will see a visual notification in the form of a red dot on the tab corresponding to the event. You should click on the tab with the red dot to see what happened.

The “Contacts” Tab

The Contacts tab shows other members of your operator group who are logged in along with how many chats each member is currently involved in (“Active Chats” column). Contacts are listed in alphabetical order.

You can start a chat with a contact by clicking on the chat Action icon for the contact:



Initiating a Chat

When you start a chat with another member of your group as described above, a 'new chat' message is sent to your contact and you'll automatically be navigated to the “Chat” tab. The Chat tab shows the chat that you are involved with at this moment – that is, the chat you just initiated:



You would normally stay on this tab to wait for answer, but you are free to navigate away from the Chat tab if you need to. If you are away when the answer is received, you'll see a tab notification to indicate that the answer has come in.

When the answer is received, on the Chat screen you see a “Connected” message and can start typing chat messages to be sent to the other user:



See the section “The Chat Tab” for further details about the Chat screen.

Receiving a Chat

When you receive a new chat, either from a member of your group or from a website visitor, you get a popup showing information about the calling user and asking if you want to take the call:



You can accept the call by clicking Confirm/Yes or reject the call by clicking Cancel/No. In the latter case, the calling user gets an indication that the user was busy.

If you accept the call and you have no other active chats, you'll automatically be navigated to the “Chat” tab. The Chat tab shows the chat that you are involved with at this moment – that is, the chat you just accepted. On the Chat screen you see the information about the calling user and a “Connected” message which means you can start typing chat messages to be sent to the other user.



See the section “The Chat Tab” for further details about the Chat screen.

If you accept the call and you do have other active chat(s), you will not be automatically navigated to the new chat. This is because you could be in the middle of typing on another chat. The new chat you just accepted will be added to the conversations list and you'll see a notification on the Conversations tab which will remind you to go there to attend to the newly accepted chat. The section on the Conversations tab describes how to get to a chat from the conversations list.

The “Chat” Tab

The Chat tab shows the chat screen as illustrated below. It shows the name of the user you're chatting with followed by the messages that have been exchanged. Below that is the area you use to send messages to the other user.

Predefined Messages

Operators can send predefined messages anytime by clicking the dropdown just above the typing box (see arrow pointer) and selecting a message from the dropdown list. If your operator group has no canned messages defined, you won't see this dropdown.

Navigation

You can keep this chat alive and still navigate away from it – to another tab and perhaps another chat (by way of the Conversations tab) – but you can always get back to this chat from the Conversations tab. If anything happens on this chat after you've navigated away from the Chat tab (such as a message received from the other user or the other user disconnects), you'll be notified and can get back to this chat to see what happened.

Disconnect & Close

Finally, the Disconnect button will disconnect the chat. You can disconnect a chat without “closing” it which keeps it in the conversations list in case you want to look back at it later, and you can close it later at any time. Alternatively, using the Close button will both disconnect and close the chat which removes it from the Chat tab and the conversations list altogether.

Chat Event Notifications

What if you're on the Chat tab chatting with one user and something happens on a different chat? For example, while you're chatting with Amit, Becky sends you a chat message or she disconnects that chat from her end? Or, suppose you just have one chat active but you're not on the Chat tab when the other user does something like send you a chat message or disconnect the chat? You need to be notified of the event.

If anything happens on a chat because of some action taken by the other user (the user you are chatting with) and you are not on the Chat tab looking at that chat when the event happens, you will see a tab notification in the form of a red dot telling you that something happened on the chat. You can click on the tab with the red dot to find out what happened.

For example: If you only deal with one chat at a time, then the Chat tab will already contain the chat having the event. In this case you'll see the red dot notification on the Chat tab:



You can then just click on the Chat tab to see what happened on the chat.

If you have more than one chat going, then there are two possibilities. If the chat currently in the Chat tab is the one that had the event, you will see the red dot notification on the Chat tab, same as above, and you just need to go there. But if the event happened on a different chat than the one currently in the Chat tab, you will see the red dot notification on the Conversations tab:



In the above example, while you are chatting with Amit on the Chat tab, some other chat had an event.

When you are ready to attend to the new event, go to the Conversations tab, and within the chat list there you'll see a red dot notification next to the chat that had the event - you can then navigate to that chat by clicking on its talk Action button:

The “Logout” Tab

Clicking on this tab prompts you for confirmation to logout. If you're ready to log out, click “Confirm”.
